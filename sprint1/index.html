    Relatório de Contribuição – Sprint 1

    Disciplina: Gestão de Configuração e Evolução de Software
    Equipe: KDE Linux
    Comunidade/Projeto de Software Livre: KDE Linux
    Período da Sprint: 20/04/2026 - 11/05/2026

    1. Objetivos da Sprint

    • [x] Identificar e receber atribuição (assign) de issues com a tag Newcomer.
    • [x] Iniciar a comunicação formal com os mantenedores no KDE Invent.
    • [x] Submeter os primeiros Merge Requests (MRs) com contribuições reais de código ou documentação.
    • [x] Estudar o codebase e tecnologias específicas (Flatpak, Hugo, Shell Script) para resolução das tarefas.

    3. Maiores Avanços

    • Contribuições Reais: A equipe avançou para contribuições diretas com múltiplos Merge Requests abertos em documentação, interface e website.
    • Comunicação Ativa: Consolidação do uso do Matrix e GitLab para alinhar propostas com os mantenedores.
    • Domínio do Fluxo: Membros conseguiram alterar o código, gerar novas imagens customizadas e validar mudanças em VMs.

    4. Maiores Dificuldades

    • Escassez de Issues: A tag Newcomer possui poucas tarefas, gerando alta concorrência interna e externa.
    • Complexidade Técnica: Necessidade de aprendizado rápido sobre Flatpak, Hugo e manipulação de arquivos STL.
    • Ciclo de Build: O processo de build local é demorado, atrasando a validação de pequenas alterações.

    5. Lições Aprendidas

    • Proatividade: A auditoria da documentação pode revelar falhas que justificam a criação de novas tarefas úteis.
    • Governança: Importância de seguir protocolos de assinatura de autoria e padrões profissionais do KDE.
    • Onboarding: A percepção de que o README é vital para evitar que novos desenvolvedores cometam erros de infraestrutura.

    6. Planejamento para a Próxima Sprint

    • [ ] Acompanhar e responder aos Code Reviews dos Merge Requests abertos.
    • [ ] Finalizar o empacotamento Flatpak para a issue de miniaturas STL.
    • [ ] Iniciar a implementação técnica das issues atribuídas nesta sprint.
    • [ ] Buscar novas tarefas de código para os integrantes que finalizaram as de documentação.
